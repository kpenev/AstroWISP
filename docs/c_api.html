bool piecewise_bicubic_fit(double **pixel_values, double **pixel_errors, char **pixel_masks, unsigned long number_images, unsigned long image_x_resolution, unsigned long image_y_resolution, char **column_names, char ***source_ids, double **column_data, unsigned long *number_sources, unsigned long number_columns, BackgroundMeasureAnnulus **backgrounds, FittingConfiguration *configuration, double *subpix_sensitivities, unsigned long subpix_x_resolution, unsigned long subpix_y_resolution, H5IODataTree *output_data_tree)¶

Fit a smoothly varying piecewise bicubic PSF model.

Parameters
  • pixel_values: The pixel values of the calibrated images to perform simultaneous fits of.
  • pixel_errors: Estimated pixel errors of the calibrated images to perform simultaneous fits of.
  • pixel_masks: Mask flags of the calibrated images to perform simultaneous fits of.
  • number_images: How many images are being simultaneously fit.
  • image_x_resolution: The common x resolution of the images being processed.
  • image_y_resolution: The common y resolution of the images being processed.
  • column_names: The names of the source columns.
  • source_ids: The IDs to assign to the sources in each image. The first index is the image index the second index is the source index within each image.
  • column_data: The values of each column for each source. The first index is the image index, the remaining array is organized as described by the column_data argument of FitPSF::IOSources::IOSources().
  • number_sources: How many sources are in column_data for each image.
  • number_columns: How many columns are used in PSF fitting.
  • backgrounds: Pointers to the measured background for the input sources indexed by the image index.
  • configuration: The configuration for how to do the fitting.
  • subpix_sensitivities: The sensitivities of the pixel at each sub-pixel part.
  • subpix_x_resolution: Subpixmap x resolution.
  • subpix_y_resolution: Subpixmap y resolution.
  • output_data_tree: A data tree to fill with the results.

void expand_term_expression(char *expansion_term_expression, char ***term_list, unsigned *num_terms)¶

Expand the given expression into a list of expressions, each defining a single PSF expansion term.

Parameters
  • expansion_term_expression: The expression to parse. See TermGenerator::Grammar for a description of the syntax.
  • term_list: The individual terms in the expression. Allocates new memory. Caller must release allocated memory when no longer required.
  • num_terms: On exit, set to the number of terms found in the term expression.

void free_term_list(char **term_list, unsigned num_terms)¶

Free the memory allocated by a previous call to expand_term_expression.

Parameters
  • term_list: The list of terms created by expand_term_expression() to free.
  • num_terms: The number of terms in term_list.

void evaluate_terms(char **term_list, unsigned num_terms, char **variable_names, double **variable_values, unsigned num_variables, unsigned num_sources, double *result)¶

Evaluate the given terms for the given list of sources.

Parameters
  • term_list: The list of terms to evaluate. Usually created by expand_term_expression().
  • num_terms: The number of terms.
  • variable_names: The names of the variables used by the expressions in terms.
  • variable_values: The values of the variables used by the expressions in terms for each source.
  • num_variables: The number of different variables used.
  • num_sources: The number of sources to evaluate the terms of.
  • result: The location to fill with the evaluated terms. Must already be allocated with a size of num_variables * num_sources. The values of all terms for the first source are at the beginning of the array, followed by the value of all terms for the second source etc.

bool query_result_tree(H5IODataTree *tree, const char *quantity, const char *format, void *result)¶

Query the result tree for a value (NULL if value is undefined).

Parameters
  • tree: The tree to extract the quantity from.
  • quantity: What quantity to get from the tree
  • format: Data type of quantity to expect. Examples: “int” for a single integer value “[int]” for an array of integers “double:double” for a pair of doubles “str” for a string entry
  • result: Destination to fill with the value of the quantity. For all formats except “str”, result must be pointer to the correct format and have sufficient space pre-allocated. For strings, result should be char**, and new memory is allocated for the string, so the caller is responsible for free-ing that.

If the value requested is found and not empty, the return value is true, otherwise, false and result is not touched.

bool get_psf_map_variables(H5IODataTree *output_data_tree, unsigned image_index, double *column_data)¶

Get the PSF map variables for a given image index from the given result tree.

Parameters
  • output_data_tree: The tree to extract the varibales from.
  • image_index: The image index for which to extract the variables.
  • column_data: The location to fill with the values of the variables. The values of each varibale are consecutive in memory. All required storage must already be allocated.

If the correspnoding entry in the tree is empty or non-existent, return false and do not touch the column_data argument.
